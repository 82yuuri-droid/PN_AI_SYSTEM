# Implementation Roadmap

## Phase 1: 見える化

目的:

- 売上、利益、工数、継続率を見える状態にする
- 感覚ではなく数字で改善判断できるようにする

成果物:

- 案件台帳テンプレート
- タスク台帳テンプレート
- 月次KPI集計フォーマット

実装候補:

- `data/projects.csv`
- `data/tasks.csv`
- `scripts/monthly_summary.py`
- `reports/YYYY-MM.md`

判断指標:

- 案件ごとの粗利が見える
- 遅延タスクが見える
- 月次レポートの素案が作れる

## Phase 2: 定型業務の自動化

目的:

- レポート、提案、進行管理の工数を削減する
- 初回継続率を上げる

成果物:

- 月次レポート生成
- 継続提案テンプレート
- 納期アラート
- 停滞タスク一覧

実装候補:

- CSVからMarkdownレポートを生成
- タスクの期限切れ、確認待ちを抽出
- 案件種別ごとの改善提案文を生成

判断指標:

- レポート作成時間が半分以下になる
- 初回案件の次月提案が必ず出る
- クライアント確認待ちが放置されない

## Phase 3: 収益改善

目的:

- 既存顧客への追加提案を増やす
- 高粗利メニューを伸ばす

成果物:

- 追加提案候補リスト
- 事業別パッケージ表
- 営業リストスコアリング

実装候補:

- 案件台帳から追加提案を抽出
- SNS運用顧客に動画制作を提案
- 採用支援顧客に採用動画、採用SNS運用を提案
- AI活用支援を全顧客に診断メニューとして提案

判断指標:

- 既存顧客からの追加売上が増える
- 平均単価が上がる
- 粗利率が改善する

## Phase 4: 商品化と再現性

目的:

- PNの提供価値をパッケージ化する
- 人員を増やさず案件数を増やせる体制にする

成果物:

- SNS運用パッケージ
- 採用支援パッケージ
- 動画制作パッケージ
- AI活用支援診断パッケージ
- 案件開始時チェックリスト
- 納品前チェックリスト

判断指標:

- 新規メンバーや外注先でも一定品質で進められる
- 案件ごとの進め方が標準化される
- 修正回数と管理工数が減る

## Codexの次アクション

1. CSVベースの案件台帳とタスク台帳を作る
2. 月次集計スクリプトを作る
3. サンプルデータで粗利、継続率、制作効率を集計する
4. Markdown形式の月次レポートを自動生成する
5. レポート内に次月の改善提案を出す
