## 1. 会社概要

### ミッション

PNの中核ミッションは「良いモノ・人・文化を埋没させない」こと。

PN OS上では、動画制作・SNS運用・クライアント管理を中枢業務として扱い、AIを使って案件受注から改善提案までを一気通貫で運営する方針が定義されている。

### 提供価値

PNの提供価値は、単にSNS投稿や動画を作ることではなく、価値ある事業・人・商品・文化を「届くべき人へ届く状態」に変えること。

具体的には以下を提供する。

- SNSを事業導線に変える
- 採用、集客、売上につながる発信設計を作る
- 映像、言葉、導線設計を組み合わせて事業成長を支援する
- クライアントの運用をAIで効率化し、継続的な改善ループを作る
- 投稿、問い合わせ、商談、受注、改善提案をデータでつなげる

### 主な顧客層

現時点で想定される主な顧客層は以下。

- SNSを事業成長に接続したい企業、経営者、個人事業主
- 採用広報、採用SNS、採用動画を強化したい企業
- 動画制作、ショート動画、SNSクリエイティブを必要とする企業
- SNS運用を内製化したい企業
- 既存の外注SNS運用に成果や再現性を感じていない企業
- AIを使って業務効率化、ナレッジ管理、レポート作成を進めたい企業

## 2. サービス

### 提供サービス一覧

PN OSと関連ページから読み取れる提供サービスは以下。

| サービス | 目的 | 収益との関係 |
| --- | --- | --- |
| SNS運用 | 認知、信用、問い合わせ、商品販売、資産化につなげる | 継続課金化しやすく、月次レポートと改善提案によりLTVを伸ばせる |
| 採用支援 | 採用SNS、採用訴求、応募導線を改善する | 採用成果が明確なため高単価化しやすい |
| 動画制作 | SNS、採用、営業、教育に使える動画を制作する | 単発売上を作りやすいが、修正回数管理が利益率を左右する |
| SNS内製化支援 | クライアント社内でSNSを回せる仕組みを作る | コンサル、研修、運用設計として高粗利化しやすい |
| AI活用支援 | 業務をAIで効率化し、再現性ある運用へ変える | PN自身の業務効率化にも転用でき、収益性が高い |
| 提案資料・LP・クリエイティブ制作 | 営業や導線の受け皿を整える | SNS運用や動画制作から追加提案しやすい |

### 各サービスの目的

SNS運用の目的:

- フォロワー数ではなく、事業成長につながる認知、信用、問い合わせを作る
- 投稿を資産化し、商材、講座、提案資料、営業導線へ転用する

採用支援の目的:

- 求人票だけでは伝わらない企業の魅力、安心材料、働く理由を言語化する
- 応募数、面談数、採用決定数につながる導線を作る

動画制作の目的:

- 価値や思想を伝わりやすい形に変換する
- SNS、採用、営業で使えるコンテンツ資産を作る

AI活用支援の目的:

- 業務を属人化から仕組み化へ移す
- 案件管理、制作進行、KPI管理、レポート、改善提案をAIで半自動化する

### 収益との関係

収益性の高い構造は「単発制作」ではなく、「運用・分析・改善提案・追加制作」が連結した状態。

収益導線:

1. SNSや動画で初回接点を作る
2. 初回相談、提案、受注へつなげる
3. 投稿・制作・分析を月次で回す
4. 成果と課題をレポート化する
5. 次月施策、動画追加、採用支援、AI活用支援を追加提案する

利益率を上げる鍵は、案件ごとの工数、修正回数、確認待ち、KPI回収を可視化すること。

## 3. ワークフロー

AI Workflowでは、案件受注から改善提案までの10工程が定義されている。

### 営業

該当工程:

- ① 案件受注
- ② クライアント情報整理

業務内容:

- 問い合わせメール、DM、紹介、商談メモを整理する
- 案件サマリーを作る
- 初回返信、見積、提案たたき台を作る
- クライアントの商材、ターゲット、目的、KPI、トーンを整理する

AI活用ポイント:

- Claude Code + Gmail MCPで問い合わせを自動要約
- ChatGPTで返信文、提案文、ヒアリング項目を作成
- Notion MCPで案件管理DB、クライアント管理DBへ登録

### 企画

該当工程:

- ③ 競合分析
- ④ 企画作成

業務内容:

- 競合アカウント、対象ジャンル、キーワードを調査する
- 伸び筋投稿、トレンド、勝ちパターンを抽出する
- 企画案、フック、切り口、タイトル候補、サムネ案を作る
- 月間コンテンツカレンダーに落とす

AI活用ポイント:

- vidIQで競合、outlier、trending、keywordを取得
- ChatGPTで企画の切り口を拡張
- Claude Codeで企画DBへ起票

### 制作

該当工程:

- ⑤ 台本作成
- ⑥ 撮影指示書作成
- ⑦ 編集指示書作成

業務内容:

- 企画から台本を作る
- フック、本編、CTA、テロップ、ナレーション原稿を作る
- 撮影指示書、カットリスト、香盤表を作る
- 編集指示書、サムネ指示、書き出し仕様を作る

AI活用ポイント:

- Claude Codeで企画DBから台本テンプレートへ展開
- ChatGPTで言い回し、フック、コピーを量産
- Google Drive MCPで素材と指示書を紐付け
- Canva MCPでサムネ、SNSクリエイティブ、提案資料を作成

### 納品

該当工程:

- ⑧ 投稿管理

業務内容:

- 完成動画、タイトル、概要、ハッシュタグ、投稿日時、投稿アカウントを管理する
- 投稿ステータスをIdea / Draft / Owner Review / Approved / Scheduled / Posted / Measuredで管理する
- 投稿URLを記録する

AI活用ポイント:

- Claude Codeで投稿パイプラインのステータス管理
- CodexでSNS APIへの自動投稿スクリプトを実装
- 承認済み投稿のみ自動投稿対象にする
- API開通までは半手動で運用する

### 分析・改善

該当工程:

- ⑨ 分析
- ⑩ 改善提案

業務内容:

- 表示回数、保存数、コメント数、DM数、問い合わせ数を記録する
- 勝ち負け要因を分解する
- 学びを記録し、次の企画へ反映する
- 改善提案書、次月企画方針、クライアント向け報告を作る

AI活用ポイント:

- CodexでKPI欠損検出、投稿後24時間の入力催促を作る
- Claude Codeで週次、月次レポートを自動生成
- ChatGPTで次施策アイデアを出す
- vidIQで次トレンドを確認する

## AI導入による改善余地

最も改善余地が大きいのは「投稿生成」ではなく、投稿後のKPI回収、学び化、次施策化、商談化、受注化の接続。

改善余地:

- 案件受注から改善提案までの各工程をNotion DBで接続する

- AI活用支援として、PN自身の仕組みをクライアントにも商品化できる

## 結論
