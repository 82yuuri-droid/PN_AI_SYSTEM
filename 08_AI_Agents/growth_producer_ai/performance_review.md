# Performance Review: PN Growth Producer AI

## Review Summary

PN Growth Producer AI is useful as a strategic drafting partner, but the first A-LIGHT test exposed a clear weakness: the system can produce polished strategy even when important data is missing.

That is dangerous. A daily AI employee must not only create good documents. It must force better inputs, identify what cannot be known yet, create measurable experiments, and improve the sales/client workflow.

## Strengths Observed

- Clear strategic framing
- Strong alignment with PN's mission
- Good recruiting SNS structure
- Useful proposal summary
- Clear content pillars and content ideas
- Good human review boundaries
- Good next-step orientation

## Weaknesses

## 1. Too Much Strategy, Not Enough Evidence

The A-LIGHT output made reasonable assumptions, but it did not force enough baseline metrics before recommending scope.

Risk:

- PN could sell or deliver based on attractive strategy without knowing the real funnel problem.

Required improvement:

- Add a baseline data intake workflow before strategy or proposal.

## 2. Weak Qualification for Proposal Readiness

The system can create a proposal summary even when budget, decision-maker, timeline, current metrics, and application funnel data are unknown.

Risk:

- Proposal quality looks high, but the deal may not be ready.

Required improvement:

- Add proposal readiness scoring and block full proposals when required data is missing.

## 3. Missing Experiment Design

The strategy suggested content directions but did not convert them into controlled experiments.

Risk:

- PN cannot learn which content actually increases applications, DMs, or profile clicks.

Required improvement:

- Add a content experiment prompt and workflow.

## 4. Missing Attribution Workflow

The system named KPIs but did not define how SNS activity connects to applications.

Risk:

- Brand awareness improves but PN cannot prove recruiting impact.

Required improvement:

- Add SNS-to-application attribution workflow.

## 5. Missing Client Interview Workflow

For recruiting strategy, the most valuable inputs come from candidate interviews, team interviews, and hiring manager interviews.

Risk:

- Content becomes generic employer branding.

Required improvement:

- Add client/candidate interview question prompts.

## 6. Missing QA and Red-Team Workflow

The AI does not yet challenge its own proposal or strategy before delivery.

Risk:

- Generic, overconfident, or unpriced recommendations may reach clients.

Required improvement:

- Add a critique prompt for proposals, SNS strategies, and client reports.

## 7. Missing Learning Loop

The system does not yet turn output performance into prompt improvements.

Risk:

- The AI employee repeats the same weaknesses.

Required improvement:

- Add output review log and prompt improvement workflow.

## Missing Workflows

- Baseline data intake
- Proposal readiness check
- SNS-to-application attribution setup
- Content experiment design
- Client/candidate interview extraction
- Strategy red-team critique
- Weekly AI output review
- Case study extraction after results

## Missing Prompts

- Baseline Data Intake
- Proposal Readiness Check
- Recruiting Interview Question Generator
- Content Experiment Designer
- Attribution Plan
- Strategy / Proposal Critic
- Case Study Extractor
- AI Output Quality Review

## Missing Data Sources

Required but missing or not yet structured:

- Current monthly applications
- Qualified application count
- Interview count
- Hire count
- Application source data
- Current SNS reach, profile visits, DMs, link clicks, saves, and video completion rate
- Current job posting conversion rate
- Current application landing page or form data
- Candidate objections and FAQs
- Hiring manager interview notes
- Employee/talent interview notes
- Approval lead time
- Content production capacity
- Previous campaign results
- Client revenue or business priority connected to hiring

## System Improvements Made

This review adds or requires:

1. Stronger system prompt rules around evidence and proposal readiness.
2. New user prompt templates for baseline data, readiness checks, experiments, attribution, critique, and learning.
3. New workflow diagrams for data intake, attribution, experimentation, and review.
4. Updated folder structure for experiments, attribution, and QA outputs.
5. Updated operating instructions so the AI employee cannot skip review and measurement.
