# Operating Instructions

## Daily Use

Use PN Growth Producer AI every business day.

Priority order:

1. Analyze new sales calls.
2. Draft or improve proposals.
3. Create follow-up messages.
4. Review client performance and suggest improvements.
5. Generate SNS strategy or content ideas.

If there is a live sales opportunity, do not start with content ideas.

## Daily Checklist

1. Check today's sales calls, DMs, referrals, and client notes.
2. Save raw inputs to `01_inbox`.
3. Run Baseline Data Intake if the client/problem is not already well understood.
4. Run the correct user prompt template.
5. Save AI output to the correct `02_outputs` folder.
6. Review output using the human review checklist.
7. Run Strategy / Proposal Critic for important external-facing outputs.
8. Send or use only the approved version.
9. Record any weak output or prompt issue in `05_review_log`.

## Weekly Operating Rhythm

## Monday: Pipeline and Proposal Focus

- Run Sales Call Analysis for all open opportunities.
- Identify deals needing proposals.
- Improve one proposal section based on objections.

## Tuesday: Outreach and Follow-Up

- Use AI to draft follow-ups.
- Use AI to convert objections into response language.
- Update the objection library.

## Wednesday: SNS Strategy and Content

- Run SNS Strategy for new clients or active prospects.
- Run Weekly Content Ideas for active clients.
- Turn priority ideas into experiments before publishing.
- Select ideas that connect to business outcomes.

## Thursday: Client Result Improvement

- Review KPI data, blockers, and deliverables.
- Run Monthly Client Improvement early for any at-risk client.
- Check attribution plan and missing data.
- Identify retention or upsell opportunities.

## Friday: Review and Asset Creation

- Review all AI outputs from the week.
- Score important outputs from 1-5.
- Save strong outputs as templates or examples.
- Run AI Output Quality Review on weak outputs.
- Update `04_knowledge`.
- Log prompt improvements.

## Monthly Operating Rhythm

1. Run Monthly Client Improvement for every active client.
2. Create renewal or upsell recommendations only when justified by client data.
3. Review AI output quality.
4. Measure time saved.
5. Review whether AI helped create revenue, retain clients, improve results, or create assets.

## Human Review Checklist

Before using AI output externally, confirm:

- The client context is accurate.
- The business problem is specific.
- The recommendation connects to money, hiring, trust, efficiency, or assets.
- The output does not overpromise.
- Pricing is reviewed by a human.
- Scope is reviewed by a human.
- Claims are supported by proof.
- The tone matches PN.
- Missing information is clearly listed.
- Facts, assumptions, and unknowns are separated.
- Measurement method is clear.
- External claims are supported by data or marked as assumptions.
- The next action is obvious.

## Output Quality Score

Score each important output from 1-5.

- 5: Ready after light edit
- 4: Useful, needs normal human edit
- 3: Directionally useful, missing important detail
- 2: Generic or risky, needs major rewrite
- 1: Do not use

If an output scores 3 or below, improve the prompt or input data before using the system again.

## Escalation Rules

Escalate to human decision immediately when:

- Pricing is unclear
- Scope is expanding
- Client asks for guarantees
- Legal or contract wording is needed
- The client is high-status or reputationally sensitive
- The AI recommends accepting a low-fit project
- The recommendation could affect client revenue, hiring, or public reputation

## Daily Minimum Standard

At minimum, each day should produce one of:

- One sales call analysis
- One proposal draft
- One follow-up message
- One SNS strategy improvement
- One client improvement recommendation
- One reusable asset from an AI output

## Definition of Done

An AI employee task is done only when:

1. Input is saved.
2. Prompt is run.
3. Output is saved.
4. Human review is completed.
5. Measurement plan is defined when relevant.
6. Next action is assigned.
7. Reusable learning is captured if useful.
